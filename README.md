# 傳說永恆

目前版本：`v0.2.6.0-alpha`

文字網頁 roguelite / 半 RPG 原型。玩家從安全據點選擇角色與地區，進行自動戰鬥冒險。現有可玩角色包含「冒險者」與可在旅途中相遇後永久解鎖的「弓箭手」；角色各自保存等級、EXP 與技能進度。死亡、撤退或通關後本輪結束，回到本輪出發據點重新整備。

## 目前內容

- 兩個正式地區：平原、森林
- 兩名可玩角色：冒險者、弓箭手
- 弓箭手會在旅途中等待玩家發現，完成相遇流程後可永久使用
- 各角色擁有獨立等級、EXP、永久成長與技能進度
- 弓箭手以精準爆擊、追擊、箭雨與毒箭建立射擊節奏，角色專屬戰鬥流程位於 `src/characters/skills/`
- 角色選單改為多角色卡片介面；未解鎖角色以未知卡呈現，查看詳情不會直接切換目前使用角色
- 新增冒險營地，主選單「開始遊戲」會先進入營地
- 完成森林普通主線後可首次抵達安平鎮，並在已造訪的安全區之間自由移動
- 安平鎮新增鐵匠鋪，可消耗金幣與素材製作四把永久武器；每種武器持有一把時不可重複製作
- 安平鎮鐵匠鋪新增正式 NPC 對話；玩家需要深入交談後才會得知鐵匠羅根的姓名，並可從對話進入既有鍛造介面
- 角色詳情新增獨立裝備頁；冒險者可使用全部武器類別，弓箭手僅能使用弓類，裝備效果會於下一輪冒險建立角色時套用
- 安全區主畫面共用同一套功能卡；點擊「移動據點」後進入獨立選擇頁，依已解鎖 Safe Area 動態列出目的地
- 冒險結束後可回到本輪出發據點，安全區可出發冒險、選擇地區、查看角色與冒險紀錄
- 新增角色等級與 EXP，擊敗敵人會累積角色經驗並自動學習技能
- 角色詳情頁可查看已學技能與下一技能的效果說明
- 存檔管理統一使用存檔碼匯出 / 匯入，方便手機與電腦間手動轉移進度
- 新增展示型倉庫與素材詳細資訊；鳳凰加護解鎖後可查看金幣、平原素材、稀有度與素材價值
- 安全區主頁整理為核心狀態、主要行動與功能格；出發冒險會先進入目前地區詳情確認本輪準備
- 鳳凰加護解鎖後可從「四處看看」前往營地去處；旅行商人支援單項數量調整與批次販賣，能將冒險素材換取金幣
- 平原與森林各有 3 項地區冒險整備；整備只影響當輪冒險並會在結算顯示貢獻
- 六項冒險整備各可投入指定素材強化；強化後仍是同一項整備，出發時會一次結算金幣與素材
- 優化物品詳細資訊排版，桌機採主要／次要雙區、手機自動切為單欄；營地進入的共用功能頁會延續營地場景
- 平原冒險與戰鬥畫面加入專屬背景，營地背景更新為新版美術
- 新增主選單、營地、平原與森林的基礎 BGM 系統，支援循環播放、淡入淡出、開關與音量保存
- 加護前死亡會失去目前等級與經驗；撤退能保留成長
- 首次擊敗平原首領後會觸發星神劇情殺，解鎖鳳凰的加護與成就系統
- 鳳凰的加護解鎖後，死亡會回到本輪出發據點並保留角色等級與經驗
- 新增逃跑與撤退流程，戰術逃跑可在同一場威脅中安全修整或反制小怪；次數用完後仍可嘗試撤離逃跑
- 撤退不算死亡，但會結束本輪並重置臨時祝福
- 主選單、地區選擇、角色選擇與成就系統皆已接入正式營地流程
- 地區與統計採列表到詳情流程；角色選擇使用可擴充的角色卡片到詳情流程
- 自動戰鬥：玩家按下一回合觀看戰報
- 戰鬥支援多名敵人同場行動，玩家可手動切換目前攻擊目標
- 戰鬥介面統一改為即時狀態、敵群卡片與戰鬥紀錄雙區配置，單敵人與多敵人共用同一套版型
- 生命條整合目前生命、回合結束預計扣血與護盾資訊；完整能力與目前祝福改為按需查看
- 戰後平原事件與三選一臨時祝福
- 目前祝福介面顯示主要流派分類，並可依攻擊、防禦、回復、負面狀態、暴擊 Tag 篩選
- 平原新增多個生存與攻擊型局中 Buff，改善第一隻精英怪前的續戰選擇
- 新增遭遇傾向系統，史萊姆 / 野獸特攻會影響後續普通與精英遭遇權重
- 調整平原精英、首領與衝鋒傷害曲線，改善第一地區通關壓力
- 修正回合結束毒傷可能讓玩家 0 HP 進入下一關的判定問題
- 新增冒險結算摘要、死因紀錄與查看最後戰鬥紀錄操作
- 調整 common / uncommon 祝福顏色，提升稀有度辨識度
- 調整戰鬥紀錄顏色，依玩家造成傷害與敵人造成傷害分色
- JS 已拆分為資料、存檔、戰鬥、UI render 與主流程模組，方便後續新增角色與地區
- 臨時祝福效果已資料化，後續新增祝福時以 `effects` 描述實際效果
- 角色與地區內容資料已搬到 JSON，方便後續用本地工具新增怪物與祝福

## 開發工具

本專案另外提供獨立內容編輯器（目前版本：`content-editor v0.1.6`），用來建立與維護地區怪物、局中 Buff / 祝福資料。工具頁不會直接修改主遊戲存檔；但可匯出工具內容包 JSON，也可匯出能直接覆蓋 `src/data/regions/*.json` 的遊戲地區 JSON。

本機啟動靜態伺服器後，可開啟：

    http://127.0.0.1:4173/tools/content-editor.html

GitHub Pages 部署後，可開啟：

    https://drinkwateru.github.io/legend-eternal-web/tools/content-editor.html

目前工具版本記錄於 `tools/VERSION`。第一版工具支援地區內容包、怪物資料、局中 Buff / 祝福、單筆複製、整包 JSON 匯入與匯出。工具頁也可以載入目前專案內建的平原地區資料，或選擇本機的 `src/data/regions/*.json` 匯入成工具內容包，方便用既有地區資料繼續新增怪物與 Buff。

## 存檔

- 目前使用瀏覽器 `localStorage` 保存基礎統計與設定。
- 存檔 key 為 `legendEternalSave`。
- 存檔資料使用英文 id，例如 `plains`、`adventurer`，避免之後顯示名稱調整時破壞存檔。
- 存檔包含 `schemaVersion`，版本更新時會盡量轉換舊資料，不直接重置。
- 統計數據頁可以用存檔碼匯出與匯入進度，方便更換裝置。
- 目前不保存本輪戰鬥狀態；關閉頁面後需要重新開始冒險。
- 統計數據頁可以刪除存檔，只會刪除本遊戲的存檔 key。

## 執行方式

建議使用專案內建的 no-cache 開發伺服器，避免瀏覽器阻擋 ES modules，也避免舊版 CSS / JS 快取干擾測試：

```bash
python tools/dev-server.py
```

接著開啟 `http://127.0.0.1:4173/`。需要改 port 時可使用 `python tools/dev-server.py --port 4174`。

需要開啟調試面板時使用：

```text
http://127.0.0.1:4173/?debug=1
```

調試面板的「安全區測試」可將據點設為已解鎖未造訪、標記並前往、直接移動、重設據點、開啟選擇據點頁，或直接播放安平鎮首次抵達。這些操作會修改目前瀏覽器的正式存檔，測試前應先匯出備份。
角色與存檔區另提供「給予鍛造資源」、「取得全部武器」與「清空全部武器」，方便測試鐵匠鋪、角色相容性與裝備流程；清空武器會同步卸下所有角色目前裝備。

## 版本與 Commit 方式

- 小型 UI、文字、數值、戰鬥節奏調整可以先留在 working tree，不一定每次 commit。
- 當完成一個可描述的階段版本時，再整理成 commit 並更新版本號。
- 版本號採 `v主版.次版.修訂.熱修-alpha`。目前仍是早期原型，因此使用 `alpha`。
- `主版` 用於正式化或大型架構變更，例如 `v1.0.0.0`。
- `次版` 用於新增明顯系統，例如等級、金幣素材、城鎮商店。
- `修訂` 用於一批可描述的小功能或 UI 改善。
- `熱修` 用於超小 bugfix，例如 Pages 快取、按鈕失效、錯字或顯示錯誤。
- Commit message 以中文描述本次階段內容，例如 `角色：實裝弓箭手技能與多角色選單`。
